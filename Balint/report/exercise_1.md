# Exercise 1: Entanglement Definitions And Probes

This section studies entanglement diagnostics in the open one-dimensional transverse-field Ising model using the project-owned exact-diagonalization path implemented in `src/nqs/`. The analysis uses an `L = 16` chain with a half-chain bipartition `A = (0, ..., 7)` and `B = (8, ..., 15)`.

The five retained field values play distinct roles in the report. `h = 0.5` is the ordered reference point, `h = 1.5` is the disordered reference point, and `h = 0.95`, `h = 1.0`, and `h = 1.05` form the near-critical cluster used to discuss the crossover region around the critical point.

## 1/a Exact Diagonalization Setup And Basis Partition

The exact-diagonalization path starts from the Hilbert-space definition in `hilbert.py`. The `SpinHilbert` class represents a spin-`1/2` space as binary arrays with entries in `{0, 1}`, provides the total dimension `2^N`, and implements the map between basis states and integer indices. Site `0` is the least-significant bit, so the basis index is

`n = sum_i s_i 2^i`.

This convention gives direct access both to specific product states and to Hamiltonian ground states in the same basis. Representative configurations such as the all-zero, all-one, and alternating-chain states can therefore be identified explicitly before solving for the exact TFIM ground state. For `L = 16`, the Hilbert-space dimension is `2^16 = 65,536`, which is still small enough for exact diagonalization and explicit reshaping of the statevector across a chosen bipartition.

The lattice geometry is defined in `graph.py`. For this exercise the relevant object is the open `Chain1D`, which provides a deterministic site ordering together with the nearest-neighbor bond list used in the Hamiltonian. This separation between Hilbert-space encoding and graph connectivity keeps the basis convention independent of the chosen interaction pattern and makes the same infrastructure reusable for both one-dimensional chains and two-dimensional square lattices elsewhere in the project.

The Hamiltonian itself is assembled in `operator.py`. The TFIM builder constructs the operator as a sum of local `sigma^z_i sigma^z_j` interaction terms over graph edges and on-site `sigma^x_i` field terms,

`H = -J sum_<ij> sigma^z_i sigma^z_j - h sum_i sigma^x_i`.

The operator representation stores these as local terms and exposes the connected matrix elements `H_{sigma', sigma}` generated by acting on a basis configuration `|sigma>`. This is important not only for exact diagonalization but also for the variational Monte Carlo estimator used later, where the local energy is written as

`E_loc(sigma) = sum_(sigma') H_(sigma, sigma') psi(sigma') / psi(sigma)`,

and the variational energy is the probability average of `E_loc(sigma)` over samples drawn from `|psi(sigma)|^2`.

The exact solver is implemented in `exact_diag.py`. It iterates over the operator matrix elements, builds a sparse COO matrix, converts it to CSR format, and then calls `scipy.sparse.linalg.eigsh` to obtain the lowest eigenpair using a Lanczos-based sparse eigensolver. The ground-state phase is fixed afterwards so that the dominant amplitude is real. This sparse construction is preferable to forming a dense `65536 x 65536` matrix and mirrors the connected-elements view that is reused later for the sampled VMC calculations.

The entanglement cut used throughout is the half-chain split `A|B` with `|A| = |B| = 8`. With this ordering the exact ground-state vector can be reshaped into a `256 x 256` amplitude matrix `psi_ab`, which is the starting point for reduced-density-matrix calculations, Schmidt decompositions, and the Renyi-2 SWAP identity used later.

## 1/b Von Neumann Entropy And The Reduced Density Matrix

For an exact pure state `|psi>`, the reduced density matrix of subsystem `A` is

`rho_A = tr_B |psi><psi|`,

and the von Neumann entropy is

`S_1(A) = -tr(rho_A log rho_A)`.

The eigenvalues of `rho_A` are the squared Schmidt coefficients across the cut. In matrix-product-state language, they quantify how much weight is carried by each Schmidt sector, so they directly determine how compressible the state is across that bond. A rapidly decaying spectrum indicates that a small bond dimension would retain most of the state weight, while a broader spectrum implies weaker compressibility.

The exact half-chain entropies are:

| `h` | Role | `S_1(A)` | `S_2(A)` |
| --- | --- | ---: | ---: |
| `0.5` | ordered reference | `0.6960` | `0.6937` |
| `0.95` | near-critical ordered side | `0.4942` | `0.3612` |
| `1.0` | critical point | `0.4234` | `0.2845` |
| `1.05` | near-critical disordered side | `0.3644` | `0.2283` |
| `1.5` | disordered reference | `0.1535` | `0.0708` |

On this finite open chain, the largest half-chain entropy occurs at `h = 0.5`, not at `h = 1.0`. The most useful reading is therefore not a single “critical point dominates everything” statement, but the full ordered-to-disordered progression together with the near-critical cluster `h = 0.95`, `1.0`, and `1.05`, which shows how the entropy changes across the crossover region.

The entanglement spectrum supports the same reading. The figure below shows the leading Schmidt weights retained from the exact diagonalization run.

![Exercise 1 entanglement spectrum](../demos/report_outputs/exercise_1/exercise_1_entanglement_spectrum.png)

The spectrum is broadest on the ordered side and becomes steadily more concentrated as the field is increased toward the disordered regime. The Schmidt-compressibility table makes this quantitative: keeping only the leading Schmidt value retains about `50.36%` of the weight at `h = 0.5`, `81.45%` at `h = 0.95`, `85.56%` at `h = 1.0`, `88.48%` at `h = 1.05`, and `96.46%` at `h = 1.5`. The disordered reference state is therefore the easiest to compress across the half-chain cut, while the ordered reference and the near-critical points require more Schmidt sectors to retain the same weight.

This also explains why `S_1` remains primarily an exact-benchmark observable in the present context. Evaluating the von Neumann entropy requires explicit access to `rho_A` or all of its eigenvalues, and that becomes exponentially expensive with subsystem size. For small exact-diagonalization benchmarks it is straightforward; for a sampled neural quantum state it is not generally an efficient default observable.

## 1/c Renyi-2 Entropy And The SWAP Identity

The Renyi entropies are defined by

`S_alpha(A) = (1 / (1 - alpha)) log tr(rho_A^alpha)`.

For `alpha = 2`, this reduces to

`S_2(A) = -log tr(rho_A^2)`,

so the key quantity is the purity `tr(rho_A^2)`. The extremal limits are simple and useful as checks. For a product state across the cut, `rho_A` is pure, so `tr(rho_A^2) = 1` and `S_2(A) = 0`. For a maximally mixed reduced density matrix on a subsystem of dimension `d_A`, `tr(rho_A^2) = 1 / d_A` and `S_2(A) = log d_A`.

The exact identity used later in the NQS setting is

`tr(rho_A^2) = <psi \otimes psi | SWAP_A | psi \otimes psi>`,

where `SWAP_A` exchanges subsystem `A` between two replicas while leaving subsystem `B` untouched. Formally, this follows from writing the reduced density matrix in the computational basis and tracing the replica product after exchanging the `A` indices. Intuitively, the estimator checks whether two copies of the state remain consistent after the `A` part is swapped. The same construction can be generalized to `alpha > 2` by introducing more replicas and a cyclic permutation, but the estimator becomes more expensive and noisier as the replica number grows.

The SWAP-check table confirms the exact identity numerically for every retained field value. For example, at `h = 1.0` the purity from `tr(rho_A^2)` is `0.7523602877`, while the purity reconstructed from `exp(-S_2)` is `0.7523602877` to machine precision. The same agreement holds at `h = 0.5`, `0.95`, `1.05`, and `1.5`, which validates the Renyi-2 route used later in the sampled neural-quantum-state setting.

Because `S_2` depends only on a low-order trace of `rho_A`, it is much more practical for Monte Carlo estimation than `S_1`. One still expects the estimator to become harder for larger subsystems, but it avoids the need to reconstruct the full reduced density matrix or all of its eigenvalues.

## 1/d Interpreting `S_1` And `S_2`

For gapped one-dimensional ground states, entanglement is expected to obey an area law: for a fixed boundary cut, `S_1(A)` and `S_2(A)` should remain `O(1)` instead of growing linearly with subsystem size. A volume-law state would instead produce entropy proportional to `|A|`. Long-range entanglement is not simply "large entropy"; it refers to nontrivial global entanglement structure that cannot be reduced to a short-range boundary contribution alone.

The subsystem-size scan is shown below.

![Exercise 1 entropy scaling](../demos/report_outputs/exercise_1/exercise_1_entropy_scaling.png)

The main finite-size message from this plot is not a clean asymptotic scaling law, but a crossover comparison. The disordered reference point `h = 1.5` stays low across subsystem sizes and is the clearest area-law-like anchor in the retained data. The ordered reference point `h = 0.5` remains the most entangled case in this finite benchmark. The near-critical trio `h = 0.95`, `1.0`, and `1.05` sits between those two limits and gives the clearest view of how the entropy profile changes across the neighborhood of criticality on an `L = 16` chain.

The heuristic log-fit table should be read cautiously. Its `R^2` values are high for `h = 0.95`, `1.0`, and `1.05`, which is consistent with enhanced finite-size growth around the crossover region, but the system is too small for strong claims about asymptotic critical scaling. The log-fit summary is therefore best read only as a compact finite-size diagnostic.

Truncating the smallest Schmidt coefficients affects `S_1` more strongly than `S_2` because the two probes weight the entanglement spectrum differently. The von Neumann entropy contains the factor `-lambda log lambda`, so even small Schmidt weights contribute. By contrast, `S_2` depends on `sum_i lambda_i^2`, which emphasizes the dominant Schmidt sectors and suppresses the tail. This is precisely why `S_1` is the more tail-sensitive diagnostic, while `S_2` is the more practical probe when one later moves to sampled neural quantum states.

## Conclusions

Exact diagonalization on the open `L = 16` TFIM chain provides a compact reference point for the entanglement probes used later in the project. The computational-basis convention, product-state access, and explicit `A|B` partition together establish a consistent setting for all subsequent observables.

The reduced-density-matrix eigenvalues define the Schmidt spectrum across the half-chain cut and therefore quantify compressibility directly in matrix-product-state language. Across the retained field values, the overall trend is a decrease in entanglement from the ordered reference `h = 0.5` toward the disordered reference `h = 1.5`, with the near-critical points `h = 0.95`, `1.0`, and `1.05` giving the most informative comparison around the crossover region.

The SWAP check verifies the Renyi-2 identity at machine precision for all five field values. This is important because `S_2` remains a practical entanglement probe in the neural-quantum-state setting, whereas `S_1` generally requires explicit reduced-density-matrix reconstruction.

The finite-size scan is most naturally interpreted by taking `h = 1.5` as the low-entanglement area-law anchor, `h = 0.5` as the high-entanglement ordered reference on this chain, and `h = 0.95`, `1.0`, `1.05` as the near-critical cluster for discussing crossover behavior. The contrast between `S_1` and `S_2` is then clear: `S_1` is more sensitive to the tail of the entanglement spectrum, while `S_2` is more robustly tied to the dominant Schmidt sectors.

## Limitations

This report is restricted to the retained exact results for a finite open `L = 16` chain and a single half-chain partition. The figures and tables should therefore be read as controlled benchmark evidence rather than proof of asymptotic scaling. In particular, the heuristic log-fit summary is only a finite-size diagnostic, and the discussion deliberately avoids treating the critical point as the single largest-entropy case because that is not what the retained outputs show for this benchmark.
